# Описание проекта

`parsing-main` — Telegram-бот на `pyTelegramBotAPI` + `Telethon` для:
- привязки пользовательского Telegram-аккаунта (MTProto-сессия);
- парсинга сообщений из выбранных каналов за заданный диапазон дат;
- генерации ответа и поста на базе собранных материалов через OpenAI.

# Установка и запуск

1. Создайте и активируйте виртуальное окружение:
```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

2. Установите зависимости:
```powershell
pip install -r requirements.txt
```

3. Создайте `.env` на основе `.env.example` и заполните значения.

4. Запустите бота:
```powershell
python bot.py
```

# Конфигурация

Основные переменные окружения:
- `BOT_TOKEN` — токен бота от BotFather.
- `TG_API_ID`, `TG_API_HASH` — доступ к Telegram API (my.telegram.org).
- `TG_FORCE_SMS` — принудительная отправка кода по SMS (`0/1`).
- `TELETHON_SESSION` — fallback string-session (если нет файла сессии пользователя).
- `OPENAI_API_KEY`, `OPENAI_MODEL`, `OPENAI_TIMEOUT_SECONDS` — настройки OpenAI.
- `AI_ENABLED`, `AI_MAX_*` — лимиты и режим генерации.

# Что реализовано

- Привязка аккаунта Telegram через номер/код/2FA.
- Хранение пользовательских данных и каналов в `storage/data.json`.
- Парсинг по списку каналов с ограничением периода (до 5 месяцев).
- Фильтрация сообщений по подстроке пользовательского запроса.
- Сохранение черновика материалов в `storage/drafts/<user_id>.json`.
- Генерация AI-ответа и AI-поста на основе собранных данных.
- Логирование авторизации в `logs/auth.log`.

# Ограничения и дальнейшие шаги

- Основная логика сосредоточена в одном файле `bot.py` (монолит), что усложняет сопровождение.
- Тесты покрывают только часть вспомогательной логики; отсутствуют интеграционные сценарии.
- Данные хранятся в JSON-файлах без блокировок и миграций.

Рекомендуемые дальнейшие шаги:
1. Вынести auth-flow и parsing-flow в отдельные модули.
2. Добавить интеграционные тесты сценариев привязки и парсинга.
3. Добавить валидацию и мониторинг ошибок на уровне production-окружения.
