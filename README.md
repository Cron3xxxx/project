# Описание проекта

`parsing-main` — Telegram-бот на `pyTelegramBotAPI` и `Telethon` для:
- привязки пользовательского Telegram-аккаунта (MTProto-сессия);
- парсинга сообщений из выбранных каналов за указанный период;
- генерации ответа и поста на базе собранных материалов через OpenAI.

# Установка и запуск

1. Создайте и активируйте виртуальное окружение:
```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

2. Установите зависимости:
```powershell
pip install -r requirements.txt
```

3. Создайте `.env` на основе `.env.example` и заполните значения.

4. Запустите бота:
```powershell
python bot.py
```

# Конфигурация

Ключевые переменные окружения:
- `BOT_TOKEN` — токен бота от BotFather.
- `TG_API_ID`, `TG_API_HASH` — доступ к Telegram API (my.telegram.org).
- `TG_FORCE_SMS` — принудительная отправка кода по SMS (`0/1`).
- `OPENAI_API_KEY`, `OPENAI_MODEL`, `OPENAI_TIMEOUT_SECONDS` — настройки OpenAI.
- `AI_ENABLED`, `AI_MAX_*` — режим и лимиты AI-генерации.

Важно:
- fallback `TELETHON_SESSION` для парсинга отключен;
- парсинг выполняется только через пользовательскую MTProto-сессию после явной привязки аккаунта.

# Что реализовано

- Привязка Telegram-аккаунта через номер/код/2FA.
- Безопасное хранение пользовательских данных в `storage/data.json`.
- Парсинг по списку каналов с ограничением периода.
- Фильтрация сообщений по пользовательскому запросу.
- Сохранение черновиков материалов в `storage/drafts/<user_id>.json`.
- Генерация AI-ответа и AI-поста на основе собранных данных.
- Логирование авторизации в `logs/auth.log` и парсинга в `logs/parsing.log`.
- Декомпозиция логики на сервисные модули (`services/*`) и покрытие unit/integration тестами (`tests/*`).

# Проверка/тестирование

Проверка синтаксиса:
```powershell
python -m compileall bot.py config.py generate_session.py remove_webhook.py services tools tests
```

Автотесты:
```powershell
python -m unittest discover -s tests -q
```

Проверяемые сценарии:
- успешная привязка аккаунта (в т.ч. 2FA);
- обработка ошибок авторизации;

- форматирование AI-ответов;
- надежность операций с JSON-хранилищем.
